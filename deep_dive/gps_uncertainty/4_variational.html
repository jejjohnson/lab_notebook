Uncertain Inputs GPs - Variational Strategies

This post is a follow-up from my previous post where I walk through the literature talking about the different strategies of account for input error in Gaussian processes. In this post, I will be discussing how we can use variational strategies to account for input error.

Posterior Approximations

What links all of the strategies from uncertain GPs is how they approach the problem of uncertain inputs: approximating the posterior distribution. The methods that use moment matching on stochastic trial points are all using various strategies to construct some posterior approximation. They define their GP model first and then approximate the posterior by using some approximate scheme to account for uncertainty. The NIGP however does change the model which is a product of the Taylor series expansion employed. From there, the resulting posterior is either evaluated or further approximated. My method actually is related because I also avoid changing the model and just attempt to approximate the posterior predictive distribution by augmenting the predictive variance function only (???).

This approach has similar strategies that stem from the wave of methods that appeared using variational inference (VI). VI consists of creating a variational approximation of the posterior distribution q(u)≈P(x)q(\mathbf u)\approx \mathcal{P}(\mathbf x)q(u)≈P(x). Under some assumptions and a baseline distribution for q(u)q(\mathbf u)q(u), we can try to approximate the complex distribution P(x)\mathcal{P}(\mathbf x)P(x) by minimizing the distance between the two distributions, D[q(u)∣∣P(x)]D\left[q(\mathbf u)||\mathcal{P}(\mathbf x)\right]D[q(u)∣∣P(x)]. Many practioners believe that approximating the posterior and not the model is the better option when doing Bayesian inference; especially for large data (example blog, VFE paper). The variational family of methods that are common for GPs use the Kullback-Leibler (KL) divergence criteria between the GP posterior approximation q(u)q(\mathbf u)q(u) and the true GP posterior P(x)\mathcal{P}(\mathbf x)P(x). From the literature, this has been extended to many different problems related to GPs for regression, classification, dimensionality reduction and more.

Model

Let's build up the GP model from the variational inference perspective. We have the same GP prior as the standard GP regression model:

P(f)∼GP(mθ,Kθ)\mathcal{P}(f) \sim \mathcal{GP}\left(\mathbf m_\theta, \mathbf K_\theta \right) P(f)∼GP(mθ​,Kθ​)

We have the same GP likelihood which stems from the relationship between the inputs and the outputs:

y=f(x)+ϵyy = f(\mathbf x) + \epsilon_y y=f(x)+ϵy​

P(y∣f,x)=∏i=1NP(yi∣f(xi))∼N(f,σy2I)\mathcal{P}(y|f, \mathbf{x}) = \prod_{i=1}^{N}\mathcal{P}\left(y_i| f(\mathbf x_i) \right) \sim \mathcal{N}(f, \sigma_y^2\mathbf I) P(y∣f,x)=i=1∏N​P(yi​∣f(xi​))∼N(f,σy2​I)

Now we just need an variational approximation to the GP posterior:

q(f)=GP(μ,ν2)q(f) = \mathcal{GP}\left( \mu, \nu^2 \right) q(f)=GP(μ,ν2)

where q(f)≈P(f∣y,X)q(f) \approx \mathcal{P}(f|y, \mathbf X)q(f)≈P(f∣y,X).

μ\muμ and ν2\nu^2ν2 are functions that depend on the augmented space Z\mathbf ZZ and possibly other parameters. Now, we can actually choose any μ\muμ and ν2\nu^2ν2 that we want. Typically people pick this to be Gaussian distributed which is augmented by some variable space Z\mathcal{Z}Z with kernel functions to move us between spaces by a joint distribution; for example:

μ(x)=k(x,Z)k(Z,Z)−1m\mu(\mathbf x) = \mathbf k(\mathbf{x, Z})\mathbf{k(Z,Z)}^{-1}\mathbf m μ(x)=k(x,Z)k(Z,Z)−1m

where θ={m,Σ,Z}\theta = \{ \mathbf{m, \Sigma, Z} \}θ={m,Σ,Z} are all variational parameters. This formulation above is just the end result of using augmented values by using variational compression (see here for more details). In the end, all of these variables can be adjusted to reduce the KL divergence criteria KL[q(f)∣∣P(f∣y,X)]\left[ q(f)||\mathcal{P}(f|y, \mathbf X)\right][q(f)∣∣P(f∣y,X)].

There are some advantages to the approximate-prior approach for example:

  • The approximation is non-parametric and mimics the true posterior.
  • As the number of inducing points grow, we arrive closer to the real distribution
  • The pseudo-points Z\mathbf ZZ and the amount are also parameters which can protect us from overfitting.
  • The predictions are clear as we just need to evaluate the approximate GP posterior.

Evidence Lower Bound (ELBO)

In VI strategies, we never get an explicit function that will lead us to the best variational approximation but we can come close; we can come up with an upper bound. Traditional marginal likelhood (evidence) function that we're given is given by:

P(y∣x,θ)⎵Evidence=∫fP(y∣f,x,θ)⎵Likelihood⋅P(f∣x,θ)⎵GP Priordf\underbrace{\mathcal{P}(y|\mathbf x, \theta)}_{\text{Evidence}}=\int_f \underbrace{\mathcal{P}(y|f, \mathbf x, \theta)}_{\text{Likelihood}} \cdot \underbrace{\mathcal{P}(f|\mathbf x, \theta)}_{\text{GP Prior}}dfEvidenceP(y∣x,θ)​​=∫f​LikelihoodP(y∣f,x,θ)​​⋅GP PriorP(f∣x,θ)​​df

where:

  • P(y∣f,x,θ)=N(y∣f,σn2I)\mathcal{P}(y|f, \mathbf x, \theta)=\mathcal{N}(y|f, \sigma_n^2\mathbf{I})P(y∣f,x,θ)=N(y∣f,σn2​I)
  • P(f∣x,θ)=N(f∣μ,Kθ)\mathcal{P}(f|\mathbf x, \theta)=\mathcal{N}(f|\mu, K_\theta)P(f∣x,θ)=N(f∣μ,Kθ​)

In this case we are marginalizing by the latent functions fff's. But we no longer consider x\mathbf xx to be uncertain with some probability distribution. Also, we do not want some MAP estimation; we want a fully Bayesian approach. So the only thing to do is marginalize out the x\mathbf xx's. In doing so we get:

P(y∣θ)=∫f∫XP(y∣f,x,θ)⋅P(f∣x,θ)⋅P(x)⋅df⋅dx\mathcal{P}(y| \theta)=\int_f\int_\mathcal{X} \mathcal{P}(y|f, \mathbf x, \theta)\cdot\mathcal{P}(f|\mathbf x, \theta) \cdot \mathcal{P}(\mathbf x)\cdot df \cdot d\mathbf{x}P(y∣θ)=∫f​∫X​P(y∣f,x,θ)⋅P(f∣x,θ)⋅P(x)⋅df⋅dx

We can rearrange this equation to change notation:

P(y∣θ)=∫X[∫fP(y∣f,x,θ)⋅P(f∣x,θ)⋅df]⎵Evidence⋅P(x)⎵Prior⋅dx\mathcal{P}(y| \theta)=\int_\mathcal{X} \underbrace{\left[ \int_f \mathcal{P}(y|f, \mathbf x, \theta)\cdot\mathcal{P}(f|\mathbf x, \theta)\cdot df\right]}_{\text{Evidence}} \cdot \underbrace{\mathcal{P}(\mathbf x)}_{\text{Prior}} \cdot d\mathbf{x}P(y∣θ)=∫X​Evidence[∫f​P(y∣f,x,θ)⋅P(f∣x,θ)⋅df]​​⋅PriorP(x)​​⋅dx

where we find that that term is simply the same term as the original likelihood, P(y∣x,θ)\mathcal{P}(y|\mathbf x, \theta)P(y∣x,θ). So our new simplified equation is:

P(y∣θ)=∫XP(y∣x,θ)⋅P(x)⋅dx\mathcal{P}(y| \theta)=\int_\mathcal{X} \mathcal{P}(y|\mathbf x, \theta) \cdot \mathcal{P}(\mathbf x) \cdot d\mathbf{x} P(y∣θ)=∫X​P(y∣x,θ)⋅P(x)⋅dx

where we have effectively marginalized out the fff's. We already know that it's difficult to propagate the x\mathbf xx's through the nonlinear functions K−1\mathbf K^{-1}K−1 and ∣|∣det K∣\mathbf K|K∣ (see previous doc for examples). So using the VI strategy, we introduce a new variational distribution q(x)q(\mathbf x)q(x) to approximate the posterior distribution P(x∣y)\mathcal{P}(\mathbf x| y)P(x∣y). The distribution is normally chosen to be Gaussian:

q(x)=∏i=1NN(x∣μz,Σz)q(\mathbf x) = \prod_{i=1}^{N}\mathcal{N}(\mathbf x|\mathbf \mu_z, \mathbf \Sigma_z) q(x)=i=1∏N​N(x∣μz​,Σz​)

So at this point, we are interested in trying to find a way to measure the difference between the approximate distribution q(x)q(\mathbf x)q(x) and the true posterior distribution P(x)\mathcal{P} (\mathbf x)P(x). Using the standard derivation for the ELBO, we arrive at the final formula:

F(q)=Eq(x)[log⁡P(y∣x,θ)]−DKL[q(x)∣∣P(x)]\mathcal{F}(q)=\mathbb{E}_{q(\mathbf x)}\left[ \log \mathcal{P}(y|\mathbf x, \theta) \right] - \text{D}_\text{KL}\left[ q(\mathbf x) || \mathcal{P}(\mathbf x) \right] F(q)=Eq(x)​[logP(y∣x,θ)]−DKL​[q(x)∣∣P(x)]

If we optimize F\mathcal{F}F with respect to q(x)q(\mathbf x)q(x), the KL is minimized and we just get the likelihood. As we've seen before, the likelihood term is still problematic as it still has the nonlinear portion to propagate the x\mathbf xx's through. So that's nothing new and we've done nothing useful. If we introduce some special structure in q(f)q(f)q(f) by introducing sparsity, then we can achieve something useful with this formulation. But through augmentation of the variable space with u\mathbf uu and Z\mathbf ZZ we can bypass this problem. The second term is simple to calculate because they're both chosen to be Gaussian.

Jensen's Inequality

This theorem is one of those sleeper theorems which comes up in a big way in many machine learning problems. I've seen it mostly in the context of optimization and variational inference.

The Jensen inequality theorem states that for a convex function fff,

E[f(x)]≥f(E[x])\mathbb{E} [f(x)] \geq f(\mathbb{E}[x]) E[f(x)]≥f(E[x])

A convex function (or concave up) is when there exists a minimum to that function. If we take two points on any part of the graph and draw a line between them, we will be above or at (as a limit) the minimum point of the graph. We can flip the signs for a concave function. But we want the convex property because then it means it has a minimum value and this is useful for minimization strategies. Recall from Calculus class 101: let's look at the function f(x)=log⁡xf(x)=\log xf(x)=logx.

We can use the second derivative test to find out if a function is convex or not. If f′(x)≥0f'(x) \geq 0f′(x)≥0 then it is concave up (or convex). I'll map out the derivatives below:

f′(x)=1xf'(x) = \frac{1}{x} f′(x)=x1​

f′′(x)=−1x2f''(x) = -\frac{1}{x^2} f′′(x)=−x21​

You'll see that −1x2≤0-\frac{1}{x^2}\leq 0−x21​≤0 for x∈[0,∞)x \in [0, \infty)x∈[0,∞). This means that log⁡x\log xlogx is a concave function. So, the solution to this if we want a convex function is to take the negative log⁡\loglog (which adds intuition as to why we typically take the negative log likelihood of many functions).

Identity Trick

I think the slides from the MLSS 2018 meeting is the only place that I have encountered anyone actually explicitly mentioning this Identity trick.

Given an integral problem:

p(x)=∫p(x∣z)p(z)dzp(x) = \int p(x|z)p(z)dz p(x)=∫p(x∣z)p(z)dz

I can multiply by an arbitrary distribution which is equivalent to 1.

p(x)=∫p(x∣z)p(z)q(z)q(z)dzp(x)=\int p(x|z) p(z) \frac{q(z)}{q(z)}dz p(x)=∫p(x∣z)p(z)q(z)q(z)​dz

Then I can regroup and reweight the integral

p(x)=∫p(x∣z)p(z)q(z)q(z)dzp(x) = \int p(x|z)\frac{p(z)}{q(z)}q(z)dz p(x)=∫p(x∣z)q(z)p(z)​q(z)dz

This results in a different expectation that we initially had

p(x)=Eq(z)[p(x∣z)p(z)q(z)]p(x) = \underset{q(z)}{\mathbb{E}}\left[ p(x|z)\frac{p(z)}{q(z)} \right] p(x)=q(z)E​[p(x∣z)q(z)p(z)​]

Sparse Gaussian Processes

Methods

  • Approximation by Subset of Data
  • Inducing Points

  • VFE

Sparse GPs refer to a family of methods that seek to take a subset of points in order to approximate the full dataset. Typically we can break them down into 5 categories:

  • Subset of Data (Transformation, Random Sampling)
  • Data Approximation Methods (Nystrom, Random Fourer Features, Random Kitchen Sinks, Sparse-Spectrum, FastFood, A la Carte)
  • Inducing Points (SoR, FITC, DTC, KISS-GP)
  • Linear Algebra (Toeplitz, Kronecker, )
  • Approximate Inference (Variational Methods)

Each of these methods ultimately augment the model so that the largest computation goes from O(N3)\mathcal{O}(N^3)O(N3) to O(MN2)\mathcal{O}(MN^2)O(MN2) where M<<NM<<NM<<N.

Subset of Data

This is the simplest way to approximate the data. The absolute simplest way is to take a random subsample of your data. However this is often not a good idea because the more data you have the more information you're more likely to have. It's an age old rule that says if you want better predictions, it's often better just to have more data.

A more sophisticated way to get a subsample of your data is to do some sort of pairwise similarity comparison scheme - i.e. Kernel methods. There are a family of methods like the Nystrom approximation or Random Fourier Features (RFF) which takes a subset of the points through pairwise comparisons. These are kernel matrix approximations so we can transform our data from our data space X∈RN×D\mathcal{X} \in \mathbb{R}^{N \times D}X∈RN×D to subset data space Z∈RM×d\mathcal{Z} \in \mathbb{R}^{M \times d}Z∈RM×d which is found through an eigen decomposition scheme.

In GPs we calculate a kernel matrix K∈RN×N\mathbf K \in \mathbb{R}^{N \times N}K∈RN×N. If NNN is large enough, then throughout the marginal likelihood, we need to calculate K−1\mathbf K^{-1}K−1 and ∣K∣|\mathbf K|∣K∣ which has O(N3)\mathcal{O}(N^3)O(N3) operations and O(N2)\mathcal{O}(N^2)O(N2) memory costs. So we make an approximate matrix K~\mathbf {\tilde{K}}K~ given by the following formula:

K~=KzKzz−1Kz⊤\tilde{K}=K_{z}K_{zz}^{-1}K_z^{\top} K~=Kz​Kzz−1​Kz⊤​

where:

  • Kzz=K(z,z)∈RM×MK_{zz}=K(z,z)\in \mathbb{R}^{M\times M}Kzz​=K(z,z)∈RM×M - the kernel matrix for the subspace Z\mathcal{Z}Z
  • Kz=K(x,z)∈RN×MK_z=K(x,z)\in \mathbb{R}^{N\times M}Kz​=K(x,z)∈RN×M - the transformation matrix from the data space X\mathcal{X}X to the subspace Z\mathcal{Z}Z
  • K≈K~∈RN×NK \approx \tilde{K} \in \mathbb{R}^{N \times N}K≈K~∈RN×N - the approximate kernel matrix of the data space X\mathcal{X}X

Below is an example of where this would be applicable where we just implement this method where we just transform the day.

from sklearn.kernel_approximation import Nystroem

Pivoting off of the method above, we So now when we calculate the log likelihood term log⁡P(y∣X,θ)\log \mathcal{P}(y|X,\theta)logP(y∣X,θ) we can have an approximation:

log⁡N(y∣0,K+σ2I)≈log⁡N(y∣0,K~+σ2I)\log \mathcal{N}(y | 0, K + \sigma^2I) \approx \log \mathcal{N}(y | 0, \tilde{K} + \sigma^2I) logN(y∣0,K+σ2I)≈logN(y∣0,K~+σ2I)

Notice how we haven't actually changing our formulation because we still have to calculate the inverse of K~\tilde{K}K~ which is RN×N\mathbb{R}^{N \times N}RN×N. Using the Woodbury matrix identity for the kernel approximation form (Sherman-Morrison Formula):

(K~+σ2I)−1=σ−2I−σ−4Kz(Kzz+σ−2Kz⊤Kz)−1Kz⊤(\tilde{K} + \sigma^2 I)^{-1}=\sigma^{-2}I - \sigma^{-4}K_z(K_{zz}+\sigma^{-2}K_z^{\top}K_z)^{-1}K_z^{\top} (K~+σ2I)−1=σ−2I−σ−4Kz​(Kzz​+σ−2Kz⊤​Kz​)−1Kz⊤​

Now the matrix that we need to invert is (Kzz+σ−2Kz⊤Kz)−1(K_{zz}+\sigma^{-2}K_z^{\top}K_z)^{-1}(Kzz​+σ−2Kz⊤​Kz​)−1 which is (M×M)(M \times M)(M×M) which is considerably smaller if M<<NM << NM<<N. So the overall computational complexity reduces to O(NM2)\mathcal{O}(NM^2)O(NM2).

Sparse GPs - Inducing Points Summary

So I think it is important to make note of the similarities between methods; specifically between FITC and VFE which are some staple methods one would use to scale GPs naively. Not only is it helpful for understanding the connection between all of the methods but it also helps with programming and seeing where each method differs algorithmically. Each sparse method is a method of using some set of inducing points or subset of data Z\mathcal{Z}Z from the data space D\mathcal{D}D. We typically have some approximate matrix Q\mathbf{Q}Q which approximates the kernel matrix K\mathbf{K}K:

Qff=KfuKuu−1Kuf\mathbf{Q}_{ff}=\mathbf{K}_{fu}\mathbf{K}_{uu}^{-1}\mathbf{K}_{uf} Qff​=Kfu​Kuu−1​Kuf​

Then we would use the Sherman-Morrison formula to reduce the computation cost of inverting the matrix K\mathbf{K}K. Below is the negative marginal log likelihood cost function that is minimized where we can see the each term broken down:

L(θ)=N2log⁡2π+12log⁡∣Qff+G∣⎵Complexity Penalty+12y⊤(Qff+G)−1y⎵Data Fit+12σn2tr(T)⎵Trace Term\mathcal{L}(\theta)= \frac{N}{2}\log 2\pi + \underbrace{\frac{1}{2} \log\left| \mathbf{Q}_{ff}+G\right|}_{\text{Complexity Penalty}} + \underbrace{\frac{1}{2}\mathbf{y}^{\top}(\mathbf{Q}_{ff}+G)^{-1}\mathbf{y}}_{\text{Data Fit}} + \underbrace{\frac{1}{2\sigma_n^2}\text{tr}(\mathbf{T})}_{\text{Trace Term}}L(θ)=2N​log2π+Complexity Penalty21​log∣Qff​+G∣​​+Data Fit21​y⊤(Qff​+G)−1y​​+Trace Term2σn2​1​tr(T)​​

The data fit term penalizes the data lying outside the covariance ellipse, the complexity penalty is the integral of the data fit term over all possible observations y\mathbf{y}y which characterizes the volume of possible datasets, the trace term ensures the objective function is a true lower bound to the MLE of the full GP. Now, below is a table that shows the differences between each of the methods.

Algorithm G\mathbf{G}G T\mathbf{T}T

Another thing to keep in mind is that the FITC algorithm approximates the model whereas the VFE algorithm approximates the inference step (the posterior). So here we just a have a difference in philosophy in how one should approach this problem. Many people in the Bayesian community will argue for approximating the inference but I think it's important to be pragmatic about these sorts of things.

Observations about the Sparse GPs

  • VFE
    • Overestimates noise variance
    • Improves with additional inducing inputs
    • Recovers the full GP Posterior
    • Hindered by local optima
  • FITC
    • Can severly underestimate the noise variance
    • May ignore additional inducing inputs
    • Does not recover the full GP posterior
    • Relies on Local Optima

Some parameter initialization strategies:

  • K-Means
  • Initially fixing the hyperparameters
  • Random Restarts

An interesting solution to find good hyperparameters for VFE:

  1. Find parameters with FITC solution
  2. Initialize GP model of VFE with FITC solutions
  3. Find parameters with VFE.

Variational Compression

Figure: This graphical model shows the relationship between the data XXX, the labels yyy and the augmented labels zzz.

This is a concept I've came across that seeks to give a stronger argument for using an augmented space Z∈RM×D\mathcal Z\in \mathbb{R}^{M \times D}Z∈RM×D instead of just the data space X∈RN×D\mathcal X \in \mathbb{R}^{N \times D}X∈RN×D. This has allowed us to reduce the computational complexity of all of our most expensive calculations from O(N3)\mathcal{O}(N^3)O(N3) to O(NM2)\mathcal{O}(NM^2)O(NM2) when we are learning the best parameters for our GP models. The term variational compression comes from the notion that we want to suppress the function valuse fff with some auxilary variables uuu. It's kind of like reducing the data space X\mathcal XX with the auxilary data space Z\mathcal ZZ in a principled way. This approach is very useful as it allows us to use a suite of variational inference techniques which in turn allows us to scale GP methods. In addition, we even have access to advanced optimization strategies such as stochastic variational inference and parallization strategies. You'll also notice that the GP literature has essentially formulated almost all major GP algorithm families (e.g. GP regression, GP classification and GP latent variable modeling) through this variation compression strategy. Below we will look at a nice argument; presented by Neil Lawrence (MLSS 2019); which really highlights the usefulness and cleverness of this approach and how it relates to many GP algorithms.

Joint Distribution - Augmented Space P(f,u)\mathcal{P}(f,u)P(f,u)

Let's add an additional set of variables uuu that's jointly Gaussian with our original function fff.

p(f,u)=N([fu];[00],[KffKfuKufKuu])p(f,u)=\mathcal{N}\left( \begin{bmatrix} f \\ u \end{bmatrix}; \begin{bmatrix} 0 \\ 0 \end{bmatrix}, \begin{bmatrix} K_{ff} & K_{fu} \\ K_{uf} & K_{uu} \end{bmatrix} \right)p(f,u)=N([fu​];[00​],[Kff​Kuf​​Kfu​Kuu​​])

We have a new space where we have introduced some auxilary variables uuu to be modeled jointly with fff. Using all of the nice properties of Gaussian distributions, we can easily write down the conditional distribution P(f∣u)\mathcal{P}(f|u)P(f∣u) and marginal distribution P(u)\mathcal{P}(u)P(u) in terms of the joint distribution P(f,u)\mathcal P (f,u)P(f,u) using conditional probability rules.

P(f,u)=P(f∣u)⋅P(u)\mathcal P (f,u) = \mathcal{P}(f|u) \cdot \mathcal{P}(u) P(f,u)=P(f∣u)⋅P(u)

where:

  • Conditional Dist.: P(f∣u)=N(f∣μu,νuu2)\mathcal{P}(\mathbf{f | u}) = \mathcal N (f| \mathbf {\mu_u, \nu^2_{uu}})P(f∣u)=N(f∣μu​,νuu2​)
    • μu=KfuKuu−1u\mu_u = \mathbf{K_{fu}K_{uu}^{-1}u}μu​=Kfu​Kuu−1​u
    • νuu2=Kff−KfuKuu−1Kuf\nu^2_{uu} = \mathbf{K_{ff} - K_{fu}K_{uu}^{-1}K_{uf}}νuu2​=Kff​−Kfu​Kuu−1​Kuf​
  • Augmented space Prior: P(u)=N(u∣0,Kuu)\mathcal P (\mathbf u) = \mathcal N\left( \mathbf u | 0, \mathbf K_{uu} \right)P(u)=N(u∣0,Kuu​)

We could actually marginalize out uuu to get back to the standard GP prior P(f)=GP(f∣m,Kff)\mathcal P (f) = \mathcal{GP} (f | \mathbf{ m, K_{ff}})P(f)=GP(f∣m,Kff​). But keep in mind that the reason why we did the conditional probability is this way is because of the computationally decreased complexity that we gain ,O(N3)→O(NM2)\mathcal{O}(N^3) \rightarrow \mathcal{O}(NM^2)O(N3)→O(NM2). We want to 'compress' the data space X\mathcal XX and subsequently the function space of fff. So now let's write the complete joint distribution which includes the data likelihood and the augmented latent variable space:

P(y,f,u∣X,Z)=P(y∣f)⎵Likelihood⋅P(f∣u,X,Z)⎵Conditional Dist.⋅P(u∣Z)⎵Prior\mathcal{P}(y,f,u|X,Z)= \underbrace{\mathcal{P}(y|f)}_{\text{Likelihood}} \cdot \underbrace{\mathcal{P} (f|u, X, Z)}_{\text{Conditional Dist.}} \cdot \underbrace{\mathcal{P}(u|Z)}_{\text{Prior}} P(y,f,u∣X,Z)=LikelihoodP(y∣f)​​⋅Conditional Dist.P(f∣u,X,Z)​​⋅PriorP(u∣Z)​​

We have a new term which is the familiar GP likelihood term P(y∣f)=N(y∣f,σy2I)\mathcal P (y|f) = \mathcal{N}(y|f, \sigma_y^2\mathbf I)P(y∣f)=N(y∣f,σy2​I). The rest of the terms we have already defined above. So now you can kind of see how we're attempting to compress the conditional distribution fff. We no longer need the prior for XXX or fff in order to obtain the joint distribution for our model. The prior we have is P(u)\mathcal P (u)P(u) which is kind of a made up variable. From henceforth, I will be omitting the dependency on XXX and ZZZ as they're not important for the argument that follows. But keep it in the back of your mind that that dependency does exist.

Conditional Distribution - P(y∣u)\mathcal{P}(y|u)P(y∣u)

The next step would be to try and condition on fff and uuu to obtain the conditional distribution of yyy given uuu, P(y∣u)\mathcal{P}(y|u)P(y∣u). We can rearrange the terms of the formula above like so:

P(y,f,u)P(u)=P(y∣f)⋅P(f∣u)\frac{\mathcal{P}(y,f,u)}{\mathcal{P}(u)}= \mathcal{P}(y|f) \cdot\mathcal{P}(f|u) P(u)P(y,f,u)​=P(y∣f)⋅P(f∣u)

and using the conditional probability rules P(A,B)=P(A∣B)⋅P(B)→P(A∣B)=P(A,B)P(B)P(A,B)=P(A|B) \cdot P(B) \rightarrow P(A|B)=\frac{P(A,B)}{P(B)}P(A,B)=P(A∣B)⋅P(B)→P(A∣B)=P(B)P(A,B)​ we can simplify the formula even further:

P(y,f∣u)=P(y∣f)⋅P(f∣u)\mathcal{P}(y,f|u)=\mathcal{P}(y|f) \cdot \mathcal{P}(f|u) P(y,f∣u)=P(y∣f)⋅P(f∣u)

So, what are we looking at? We are looking at the new joint distribution of yyy and fff given the augmented variable space that we have defined. One step closer to the conditional density. In the nature of GP models and Bayesian inference in general, the next step is to see how we obtain the marginal likelihood where we marginalize out the fff's. In doing so, we obtain the conditional density that we set off to explore:

P(y∣u)=∫fP(y∣f)⋅P(f∣u)⋅df\mathcal{P}(y|u)=\int_f \mathcal{P}(y|f) \cdot \mathcal{P}(f|u) \cdot df P(y∣u)=∫f​P(y∣f)⋅P(f∣u)⋅df

where:

  • P(y∣f)\mathcal{P}(y|f)P(y∣f) - Likelihood
  • P(f∣u)\mathcal{P}(f|u)P(f∣u) - Conditional Distribution

The last step would be to try and see if we can calculate P(y)\mathcal{P}(y)P(y) because if we can get a distribution there, then we can actually train our model using marginal likelihood. Unfortunately we are going to see a problem with this line of thinking when we try to do it directly. If I marginalize out the uuu's I get after grouping the terms:

P(y)=∫u[∫fP(y∣f)⋅P(f∣u)⋅df]⎵P(y∣u)⋅P(u)⋅du\mathcal{P}(y)=\int_u \underbrace{\left[\int_f \mathcal{P}(y|f) \cdot \mathcal{P}(f|u) \cdot df \right]}_{\mathcal{P}(y|u)} \cdot \mathcal P(u) \cdot du P(y)=∫u​P(y∣u)[∫f​P(y∣f)⋅P(f∣u)⋅df]​​⋅P(u)⋅du

which reduces to:

P(y)=∫uP(y∣u)⋅P(u)⋅du\mathcal{P}(y)=\int_u \mathcal{P}(y|u) \cdot \mathcal P(u) \cdot du P(y)=∫u​P(y∣u)⋅P(u)⋅du

This looks very similar to the parameter form of the marginal likelihood. And technically speaking this would allow us to make predictions by conditioning on the trained data P(y∗∣y)\mathcal P (y*|y)P(y∗∣y). The two important issues are highlighted in that equation alone:

  1. We now have the same bottleneck on our parameter for uuu as we do for standard Bayesian parametric modeling.
  2. The computation of P(y∣u)\mathcal P (y|u)P(y∣u) is not trivial calculation and we do not get any computational complexity gains trying to do that integral with the prior P(u)\mathcal P (u)P(u).

Variational Bound on P(y∣u)\mathcal P (y|u)P(y∣u)

We've shown the difficulties of actually obtaining the probability density function of P(y)\mathcal{P}(y)P(y) but in this section we're just going to show that we can obtain a lower bound for the conditional density function P(y∣u)\mathcal{P}(y|u)P(y∣u)

P(y∣u)=∫fP(y∣f)⋅P(f∣u)⋅df\mathcal{P}(y|u)=\int_f \mathcal P (y|f) \cdot \mathcal{P}(f|u) \cdot df P(y∣u)=∫f​P(y∣f)⋅P(f∣u)⋅df

I'll do the 4.5 classic steps in order to arrive at a variational lower bound:

  1. Given an integral problem, take the log⁡\loglog of both sides of the function.

log⁡P(y∣u)=log⁡∫fP(y∣f)⋅P(f∣u)⋅df\log \mathcal P (y|u) = \log \int_f \mathcal P (y|f) \cdot \mathcal{P}(f|u) \cdot df logP(y∣u)=log∫f​P(y∣f)⋅P(f∣u)⋅df

  1. Introduce the variational parameter q(f)q(f)q(f) as a proposal with the Identity trick.

log⁡P(y∣u)=log⁡∫fP(y∣f)⋅P(f∣u)⋅q(f)q(f)⋅df\log \mathcal P (y|u) = \log \int_f \mathcal P (y|f) \cdot \mathcal{P}(f|u) \cdot \frac{q(f)}{q(f)} \cdot df logP(y∣u)=log∫f​P(y∣f)⋅P(f∣u)⋅q(f)q(f)​⋅df

  1. Use Jensen's inequality for the log function to rearrange the formula to highlight the importance weight and provide a bound for F(q)\mathcal{F}(q)F(q):

L()=log⁡P(y∣u)≥∫fq(f)⋅log⁡P(y∣f)⋅P(f∣u)q(f)⋅df=F(q)\mathcal L () = \log \mathcal P (y|u) \geq \int_f q(f) \cdot \log \frac{\mathcal P (y|f) \cdot \mathcal{P}(f|u)}{q(f) } \cdot df = \mathcal F (q) L()=logP(y∣u)≥∫f​q(f)⋅logq(f)P(y∣f)⋅P(f∣u)​⋅df=F(q)

  1. Rearrange to look like an expectation and KL divergence using targeted log⁡\loglog rules:

F(q)=∫fq(f)⋅log⁡P(y∣f)⋅df−∫fq(f)⋅log⁡P(f∣u)q(f)⋅df\mathcal F (q) = \int_f q(f) \cdot \log \mathcal P(y|f) \cdot df - \int_f q(f) \cdot \log \frac{\mathcal{P}(f|u)}{q(f)} \cdot df F(q)=∫f​q(f)⋅logP(y∣f)⋅df−∫f​q(f)⋅logq(f)P(f∣u)​⋅df

  1. Simplify notation to look like every paper in ML that uses VI to profit and obtain the variational lower bound.

F(q)=Eq(f)[log⁡P(y∣f)]−DKL[q(f)∣∣P(f∣u)]\mathcal F (q) = \mathbb E_{q(f)} \left[ \log \mathcal P(y|f) \right] - \text{D}_{\text{KL}} \left[ q(f) || \mathcal{P}(f|u)\right] F(q)=Eq(f)​[logP(y∣f)]−DKL​[q(f)∣∣P(f∣u)]

Titsias Innovation: et q(f)=P(f∣u)q(f) = \mathcal{P}(f|u)q(f)=P(f∣u).

log⁡P(y∣u)≥∫fP(f∣u)⋅log⁡P(y∣f)⋅df\log \mathcal P (y|u) \geq \int_f \mathcal P (f|u) \cdot \log \mathcal P(y|f) \cdot df logP(y∣u)≥∫f​P(f∣u)⋅logP(y∣f)⋅df

Typically we will have some matrix K∈RN×N\mathbf K\in \mathbb R^{N\times N}K∈RN×N which implies we need to calculate the inverse K−1\mathbf K^{-1}K−1 and the determinant ∣|∣det K∣\mathbf K|K∣ which both require O(N3)\mathcal{O}(N^3)O(N3).
